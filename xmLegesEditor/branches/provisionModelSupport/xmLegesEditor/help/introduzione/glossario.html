Glossario

Elemento della struttura formale
una porzione di testo con un ruolo all'interno del documento (es: articolo, comma. riferimento normativo). Generalmente gli elementi della struttura formale sono descritti dagli standard NIR e navigabili all'interno della visualizzazione struttura di xmLegesEditor;

Attributi di un elemento della struttura formale
informazioni aggiuntive o normalizzate relative al contenuto di un elemento della struttura formale (es: un elemento "data" può avere una forma testuale variabile ("gg/mm/aaaa", "gg Mese aaaa", ecc.) in accordo con le regole di drafting legislativo, ma generalmente deve anche essere descritta all'interno di un attributo dell'elemento "data" in forma normalizzata ("aaaammgg");

URN di un documento NIR
identificatore univoco di un documento NIR.

Validità di un documento
In ambito XML si dice che un documento è valido se rispetta la conformità alle regole sintattiche e strutturali definite in un particolare file di regole (DTD (Document Type Definition) o XMLSchema), definite come standard nell'ambito di un progetto. Tali regole definiscono una particolare tipologia di documenti. In ambito NIR (e quindi in xmLegesEditor) si dice che un documento è valido se è conforme agli standard DTD definiti dal progetto NIR.
